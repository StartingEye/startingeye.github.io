서버 운영과 보안의 표준인 **리눅스(Linux)** 의 아키텍처를 이해하고, 단순 조회를 넘어 시스템을 실질적으로 제어하고 관리하기 위한 심화 명령어들을 학습했습니다.

## 1. 리눅스의 핵심 구성 요소
리눅스는 단순한 소프트웨어가 아니라 하드웨어와 사용자 사이를 연결하는 정교한 계층 구조를 가집니다.

* **커널(Kernel)**: 리눅스의 심장입니다. 하드웨어를 직접 제어하고 프로세스, 메모리, 파일 시스템을 관리합니다.
* **쉘(Shell)**: 사용자의 명령을 해석하여 커널에 전달하는 번역기 역할을 합니다. (예: bash, zsh)
* **파일 시스템**: 리눅스는 "모든 것을 파일로 처리"합니다. 장치(Device)조차도 `/dev` 아래의 파일로 인식됩니다.



---

## 2. 관리자를 위한 리눅스 필수 명령어 

단순한 파일 목록 확인을 넘어, 시스템 상태를 파악하고 제어하는 명령어들입니다.

### 📂 파일 및 디렉토리 관리

| 명령어 | 상세 기능 | 활용 예시 |
| :--- | :--- | :--- |
| **ls -al** | 숨겨진 설정 파일(.)을 포함한 모든 파일의 상세 권한 확인 | `ls -al /etc` |
| **cp -r** | 디렉토리를 내부 파일까지 포함하여 통째로 복사 | `cp -r origin_dir backup_dir` |
| **rm -rf** | 경로 내의 모든 파일을 묻지 않고 강제로 삭제 (※주의 필요) | `sudo rm -rf /tmp/test` |
| **find** | 특정 조건의 파일을 시스템 전체에서 검색 | `find / -name "*.conf"` |



### ⚙️ 시스템 상태 및 권한 제어


| 명령어 | 상세 기능 | 활용 예시 |
| :--- | :--- | :--- |
| **chmod** | 파일이나 디렉토리의 읽기/쓰기/실행 권한 변경 | `chmod 755 script.sh` |
| **chown** | 파일의 소유자나 그룹을 변경 | `sudo chown root:root file` |
| **grep** | 파일 내에서 특정 문자열이 포함된 라인만 추출 | `cat access.log | grep "404"` |

## 3. 보안전문가 입장에서 주목해야 할 디렉토리 구조
리눅스 파일 시스템(FHS) 표준에 따라 보안상 중요한 데이터들이 특정 위치에 고정되어 있습니다.

* **`/etc/shadow`**: 사용자 비밀번호가 암호화되어 저장되는 파일입니다. (root만 접근 가능)
* **`/var/log/auth.log`**: 로그인 시도 및 인증 관련 로그가 기록되어 침입 탐지의 핵심이 됩니다.
* **`/tmp`**: 모든 사용자가 쓰기 권한을 가지는 임시 공간으로, 종종 악성코드의 실행 경로로 이용되기도 합니다.
* **`/proc`**: 실행 중인 프로세스 정보가 가상 파일 형태로 존재하며, 시스템 공격 및 방어 시 프로세스 추적에 사용됩니다.

## 💡 느낀 점 & 보안 포인트
* **최소 권한의 원칙**: 모든 작업을 `root`로 수행하기보다, 일반 계정에서 필요한 경우에만 `sudo`를 사용하는 습관이 시스템 파괴를 막는 첫걸음임을 배웠습니다.
* **절대 경로 vs 상대 경로**: 스크립트 작성 시 경로 오인으로 인한 보안 허점을 방지하기 위해 절대 경로 사용을 생활화해야겠습니다.

## 🚀 앞으로의 계획

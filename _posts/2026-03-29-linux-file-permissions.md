# 1. 권한 표시 읽는 법과 체크 흐름
리눅스의 모든 파일과 디렉터리는 **소유자(User)**, **소유 그룹(Group)**, **기타(Other)** 세 범주로 나뉘어 각각 읽기(`r`), 쓰기(`w`), 실행(`x`) 권한이 부여됩니다.

```bash
ls -l
# 출력 예: -rwxr-xr-x 1 root root ...
```

**[권한별 의미 요약]**

| 기호 | 영문 | 파일에서의 의미 | 디렉터리에서의 의미 |
| :---: | :---: | :--- | :--- |
| `r` (4) | Read | 파일 내용 읽기 (`cat`) | 목록 확인 (`ls`) |
| `w` (2) | Write | 파일 내용 수정 | 디렉터리 내 파일 생성 및 삭제 |
| `x` (1) | eXecute | 프로그램/스크립트로써 실행 | 권한 체크 및 디렉터리 진입 (`cd`) |

# 2. 기본 권한 및 소유권 변경 (`chmod`, `chown`)

## 2.1 파일 권한 변경: `chmod`
권한을 변경할 때는 숫자로 빠르게 계산하는 **8진수 표기법**과 특정 대상에만 적용하는 **기호 표기법**을 활용합니다.

**① 8진수 표기법** (`r=4`, `w=2`, `x=1`을 더함)
```bash
chmod 600 my_file.txt  # 나(소유자)만 읽기+쓰기, 나머지는 차단 (rw-------)
chmod 644 readme.txt   # 나는 읽기+쓰기, 남들은 읽기만 가능 (rw-r--r--)
chmod 755 run.sh       # 나는 전체 권한, 남들은 읽기+실행 (rwxr-xr-x)
```

**② 기호 표기법** (`u, g, o, a` 와 `+, -, =` 조합)
```bash
chmod u+x run.sh       # 소유자(user)에게 실행(x) 추가(+)
chmod g-w file.txt     # 소유그룹(group)에서 쓰기(w) 제거(-)
```


## 2.2 소유권 변경: `chown`
파일의 소유자나 소유 그룹 자체를 아예 변경할 때는 `chown` 명령어를 사용합니다. (root 권한 필요)
```bash
sudo chown mango file.txt          # 소유자를 mango로 변경
sudo chown mango:animals file.txt  # 소유자는 mango, 소유그룹은 animals로 동시 변경
```

# 3. 치명적인 보안 사고를 막는 3대 특수 권한 비트
단순한 `rwx` 기본 권한 외에도 시스템 보안과 팀 협업 환경(공용 폴더) 관리를 위해 꼭 알아야 하는 특별한 설정들이 있습니다.

## 3.1 Sticky bit (`t`) - 공용 폴더의 안전장치
공용 폴더에 쓰기 권한을 열어주면 다른 동료가 작성한 중요한 파일도 지워버릴 수 있는 위험이 따릅니다. **Sticky bit**를 디렉터리에 설정하면 <u>해당 디렉터리 안에서는 본인이 생성한 파일만 본인이 삭제</u>할 수 있도록 안전장치가 걸립니다.
```bash
sudo chmod +t /tmp/farm 
# 기타(other) 권한 제일 끝 x 자리가 t나 T로 바뀜 (예: drwxrwxrwt)
```


## 3.2 SUID (`s`) - 소유자 권한 대여
실행 파일에 설정하면, 일반 사용자가 해당 파일을 실행할 때 일시적으로 **파일 원본 소유자의 권한(보통 root)으로 동작**하게 해줍니다.
리눅스의 사용자 비밀번호를 변경하는 `/usr/bin/passwd`가 대표적인 예시입니다.
```bash
chmod 4755 file  # 또는 chmod u+s file
# 소유자의 권한표시 x 자리가 s 문자로 변경됨 (예: -rwsr-xr-x)
```

## 3.3 SGID (`s`) - 그룹 권한 상속
주로 협업 디렉터리에 적용합니다. 이곳에 파일을 새로 생성하면, 제작한 사람의 기본 그룹을 따라가는 것이 아니라 **부모 디렉터리가 가진 소유 그룹을 무조건 상속**받게 되어 팀원 간 파일 접근 권한 불일치 문제를 해결합니다.
```bash
chmod 2775 dir   # 또는 chmod g+s dir
# 소유그룹의 권한표시 x 자리가 s 문자로 변경됨 (예: drwxrwsr-x)
```

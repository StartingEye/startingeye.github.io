## 📅 오늘 학습한 내용
네트워크 통신의 표준 가이드라인인 **OSI 7계층 모델**과 실제 인터넷의 기반이 되는 **TCP/IP 프로토콜 스택**, 그리고 각 단계에서 사용되는 **주소 지정(Addressing)** 체계를 학습했습니다.

---

## 1. 프로토콜 계층 구조의 원리
통신 시스템이 복잡해짐에 따라 전체 임무를 여러 개의 계층으로 나누어 관리합니다.
* **계층화된 통신**: 각 계층은 자신의 바로 아래 계층이 제공하는 서비스를 이용합니다.
* **독립성**: 하드웨어나 소프트웨어 기반의 논리적 변화에 관계없이 서로 다른 시스템 간 원활한 통신을 가능하게 합니다.

---

## 2. OSI 7계층 모델 (OSI Basic Reference Model)
ISO에서 제정한 모델로, 모든 종류의 컴퓨터 시스템 간 통신을 가능하게 하는 논리적 구조입니다.

| 계층 (Layer) | 주요 역할 및 기능 | 데이터 단위 |
| :--- | :--- | :--- |
| **7. 응용 (Application)** | 사용자 소프트웨어의 네트워크 접근 및 서비스 제공 | 메시지 |
| **6. 표현 (Presentation)** | 데이터 암호화, 압축, 구문 및 의미 변환 담당 | 메시지 |
| **5. 세션 (Session)** | 통신 시스템 간 대화 설정, 유지 및 동기화 책임 | 메시지 |
| **4. 전송 (Transport)** | 프로세스-대-프로세스 전달 및 오류 복구 | 세그먼트 |
| **3. 네트워크 (Network)** | 패킷을 발신지에서 목적지까지 전달 및 경로 지정 | 데이터그램 |
| **2. 데이터 링크 (Data Link)** | 물리층을 신뢰할 수 있는 링크로 변환, 프레임 구성 | 프레임 |
| **1. 물리 (Physical)** | 비트 스트림을 물리 매체를 통해 실제로 전송 | 비트 |

## 3. TCP/IP 프로토콜 그룹
OSI 모델보다 먼저 개발되었으며, 현재 인터넷에서 실제로 사용되는 계층 구조입니다.
* **구조**: 응용, 전송, 네트워크, 데이터 링크, 물리층의 5개 계층으로 주로 구성됩니다.
* **특징**: 여러 개의 작은 네트워크(링크)들이 라우터나 교환기로 연결된 형태를 가집니다.

---

## 4. 4가지 주소 지정 방식 (Addressing)
계층에 따라 사용하는 주소가 다르며, 데이터가 목적지까지 정확히 도달하게 돕습니다.

1. **물리 주소 (Physical Address)**: NIC에 부여된 6바이트(48비트) 주소로, 동일 링크 내 노드 식별에 사용됩니다.
2. **논리 주소 (Logical/IP Address)**: 전 세계 인터넷 호스트를 식별하는 32비트 주소로, 목적지까지 변하지 않습니다.
3. **포트 주소 (Port Address)**: 컴퓨터 내 실행 중인 특정 프로세스를 식별하는 16비트 주소입니다.
4. **응용-지정 주소**: 이메일 주소나 URL처럼 특정 응용을 위해 설계된 주소입니다.



---

## 💡 느낀 점 & 보안 포인트
* **캡슐화(Encapsulation)**: 데이터가 하위 계층으로 내려가며 헤더가 붙는 과정을 통해 통신이 구조화됨을 이해했습니다.
* **보안 전공자 관점**: IP 스푸핑이나 MAC 공격을 방어하기 위해 각 계층의 주소 체계를 완벽히 숙지하는 것이 보안의 중요점 임을 느꼈습니다.
* **표현층의 역할**: 암호화와 압축이 표현층에서 일어난다는 점을 기억하여 보안 설계 시 참고해야겠습니다.

* **네트워크 보안 심화**: 각 계층별 대표적인 취약점(ARP 스푸핑, TCP SYN 플러딩 등)에 대해 공부하고 실습 환경을 구축해 볼 예정입니다.

## 2025년 2회 3과목: 프로그래밍 일반

**41. 다음 중 선점 스케줄링 알고리즘이 아닌 것은?**
1. RR(Round Robin)
2. SRT(Shortest Remaining Time)
3. HRN(Highest Response-ratio Next)
4. MQ(Multi-level Queue)

> **정답**: 3
> **해설**: CPU 스케줄링 운영 체제 기법 중 라운드 로빈(RR), SRT, 다단계 큐(MQ) 등은 프로세스가 끝나기 전에도 CPU를 강제로 뺏을 수 있는 **선점형(Preemptive)**입니다. FCFS, SJF, **HRN** 등은 작업이 끝날 때까지 CPU를 독점하는 **비선점형**입니다.

<br/>

**42. 다음 그림과 같은 기억장소에서 16K를 요구하는 프로그램이 두 번째 공백인 16K의 작업 공간에 배치되는 기억장치 배치 전략은?**
(공백 리스트: 30K -> 16K -> 50K)
1. First Fit
2. Worst Fit
3. Best Fit
4. Second Fit

> **정답**: 3
> **해설**: 16K 크기를 가장 낭비 없이 정확히 딱 맞춰 넣기 위해 탐색하여 16K 공간에 할당했으므로 단편화를 제일 적게 남기는 **최적 적합(Best Fit)** 기법에 해당합니다. 

<br/>

**43. 객체지향 프로그래밍 언어가 절차지향 프로그래밍 언어에 비해 특히 우수한 점은?**
1. 구조화 프로그래밍이 가능하다.
2. 함수를 자유자재로 사용할 수 있다.
3. 컴파일 시 실행 파일의 속도가 향상된다.
4. 유지보수성과 재사용성이 좋다.

> **정답**: 4
> **해설**: 클래스(Class)와 객체, 상속 기능을 활용하여 모델링하는 객체지향 언어의 가장 큰 장점은 모듈화된 기능을 다시 조립해 쓰기 편하여 **소프트웨어 작성의 재사용성과 유지/보수의 안정성**이 극대화된다는 점입니다.

<br/>

**44. BNF 표기법에서 정의를 나타내는 기호는?**
1. >>
2. <<
3. =#
4. ::=

> **정답**: 4
> **해설**: 문맥 자유 문법을 나타내는 배커스-나우어 표기법(BNF)에서 "정의된다(is defined as)"의 뜻으로 관습적으로 사용하는 기호는 **`::=`** 입니다.

<br/>

**45. 컴파일러의 단계에서 코드 최적화는 개선 대상에 따라 그 분류가 다양하다. 코드 최적화의 분류에 속하지 않는 것은?**
1. 색인 최적화
2. 지역 최적화
3. 전역 최적화
4. 프로시저 간 최적화

> **정답**: 1
> **해설**: 코드의 불필요한 계산을 줄여 속도를 높이는 최적화 과정은 적용 범위에 따라 하나의 기본 블록 내의 **지역(Local)**, 전체 흐름 그래프 구조를 보는 **전역(Global)**, 프로시저 구조에 대한 최적화 등으로 나뉩니다. 색인(인덱스) 최적화라는 분류는 컴파일러 이론에 흔히 존재하지 않습니다.

<br/>

**46. 다음 중 교착 상태를 배제(예방)하기 위한 방법으로 옳지 않은 것은?**
1. 하나의 자원을 점유하고 있으면서 다른 프로세스에 할당되어 사용되고 있는 자원을 추가로 점유할 수 없도록 한다.
2. 다른 프로세스에 할당된 자원은 사용이 끝날 때까지 강제로 빼앗을 수 없도록 한다.
3. 한 번에 한 개의 프로세스만이 공유 자원을 사용할 수 있도록 한다.
4. 공유 자원과 공유 자원을 사용하기 위해 대기하는 프로세서들이 원형으로 구성되는 환형 대기를 용인한다.

> **정답**: 4
> **해설**: 교착 상태를 예방하기 위해서는 교착 상태의 4가지 선결조건(상호배제, 대기, 비선점, 환형대기) 중 최소 하나를 부정/파괴해야 합니다. 따라서 환형 대기는 용인하는 것이 아니라 **허용하지 않도록 차단**해야 합니다.

<br/>

**47. 루트로부터 터미널 노드 쪽으로 파스 트리를 구성하는 하향식(Top-Down) 파서에 해당하는 것은?**
1. Shift/Reduce Parser
2. LR Parser
3. Recursive Descent Parser
4. Precedence Parser

> **정답**: 3
> **해설**: 최상위 루트 노드부터 아래로 내려가며 문법적인 구조를 분해하는 하향식 파서의 대표적인 종류로 **재귀 하향 구문 분석기(Recursive Descent Parser)**가 있습니다. LR 파서 등은 상향식입니다.

<br/>

**48. 요소 선택과 삭제는 한쪽에서, 삽입은 다른 쪽에서 일어나도록 제한하는 자료구조는?**
1. 스택
2. 트리
3. 큐
4. 방향 그래프

> **정답**: 3
> **해설**: 삽입 한쪽에서 삭제는 반대쪽에서 발생하여, 들어온 순서대로 먼저 나가는(FIFO) 선입선출 방식을 유지하는 선형 자료구조는 **Queue(큐)**입니다.

<br/>

**49. 인터프리터 기법에 관한 설명으로 옳은 것은?**
1. 반복 작업이 많은 프로그램에 적절하다.
2. 소프트웨어로 시뮬레이션하는 방법으로 사용하기에 적절하다.
3. 목적 언어 프로그램을 출력한다.
4. 실행 효율성이 높다.

> **정답**: 2
> **해설**: 인터프리터(Interpreter)는 매번 읽으며 실행(번역)하므로 속도는 느리지만 사용자와 대화형 처리나 **시뮬레이션을 통한 즉각적인 수정 및 결과 피드백**에는 훌륭하게 대응할 수 있습니다. 반면 목적 언어(기계어 파일)는 별도로 남기지 않습니다.

<br/>

**50. 다음 C 프로그램 문장에서 토큰(Token)의 개수는?**
`a[index]=1+2;`
1. 3
2. 4
3. 6
4. 9

> **정답**: 4
> **해설**: 문장을 어휘 분석기가 쪼개는 어휘 항목을 세어 보면 `a`, `[`, `index`, `]`, `=`, `1`, `+`, `2`, `;` 로 각각 분리됩니다. 총 9개의 토큰입니다.

<br/>

**51. 묵시적 제어에 관한 설명으로 가장 적절한 것은?**
1. 프로그램 사이의 자료 전달을 제어
2. 미리 지정된 순서를 프로그래머가 직접 변경하는 것
3. 자료 제어에 관한 방법
4. 프로그래머가 직접 제어하지 않는 경우 미리 정해진 순서대로 제어

> **정답**: 4
> **해설**: 묵시적 제어(Implicit Control)는 개발자가 `if`나 `goto`와 같이 흐름 제어 지시를 적지 않을 때, 컴파일러가 기본적으로 정해둔 규칙(**순차적 흐름**)에 의해 수행이 넘어가는 것을 의미합니다.

<br/>

**52. C 언어의 do ~ while 문에 대한 설명 중 틀린 것은?**
1. 문의 조건이 거짓인 동안 루프 처리를 반복한다.
2. 문의 조건이 처음부터 거짓일 때도 문을 최소 한 번은 실행한다.
3. 무조건 한 번은 실행하고 경우에 따라서는 여러 번 실행하는 처리에 사용하면 유용하다.
4. 문의 맨 마지막에 “;”이 필요하다.

> **정답**: 1
> **해설**: while 문과 동일하게 논리식이 **'참(True)'인 동안** 블록 내부를 지속해서 반복 처리합니다. 거짓이 되면 루프를 탈출합니다.

<br/>

**53. 형식 문법에서 Type 1 문법을 인식하는데 사용되는 인식기는?**
1. Finite Automata
2. Push Down Automata
3. Linear Bounded Automata
4. Turing Machine

> **정답**: 3
> **해설**: 촘스키 계층 구조에서 정규 문법(Type 3)은 유한 오토마타(FA), 문맥 자유 문법(Type 2)은 푸시다운 오토마타(PDA), **문맥 의존 문법(Type 1)은 선형 한계 오토마타(LBA, Linear Bounded Automata)**, 그리고 무제한 문법(Type 0)은 튜링 머신(TM)이 인식합니다.

<br/>

**54. 다음 중 프로그램 수행 순서로 옳은 것은?**
㉠ 컴파일러  ㉡ 로더  ㉢ 링커
1. ㉡ → ㉠ → ㉢
2. ㉠ → ㉡ → ㉢
3. ㉠ → ㉢ → ㉡
4. ㉢ → ㉠ → ㉡

> **정답**: 3
> **해설**: 원시 소스를 목적파일로 번역하는 **컴파일(㉠)** -> 목적 파일과 라이브러리를 연결하는 **링커(㉢)** -> 완성된 실행파일을 주기억장치에 올리는 **로더(㉡)** 흐름으로 진행됩니다.

<br/>

**55. 페이징 시스템에서 페이지의 크기에 관한 설명으로 옳지 않은 것은?**
1. 페이지의 크기가 작을수록 보다 적절한 작업세트를 유지할 수 있다.
2. 페이지의 크기가 클수록 참조되는 정보와 무관한 정보들이 많이 적재된다.
3. 페이지의 크기가 클수록 내부 단편화가 감소한다.
4. 페이지의 크기가 작을수록 페이지 테이블의 크기가 커진다.

> **정답**: 3
> **해설**: 쪼개는 단위인 프레임(페이지)의 블록 덩어리가 크면, 그 안에 미처 다 채우지 못하고 남는 공간(내부 단편화) 역시 전체 용량 대비 **증가**하게 됩니다.

<br/>

**56. 시스템 타이머에서 일정한 시간이 만료된 경우나 오퍼레이터가 콘솔상의 인터럽트 키를 입력한 경우 발생하는 인터럽트는?**
1. 입출력 인터럽트
2. 외부 인터럽트
3. SVC 인터럽트
4. 프로그램 검사 인터럽트

> **정답**: 2
> **해설**: 프로그램 내부의 실행 연산과 무관하게, 타이머 만료나 사용자가 콘솔 스위치를 눌렀을 때 CPU 외적인 요인에서 거는 인터럽트는 **외부(External) 인터럽트**에 속합니다.

<br/>

**57. 프로그램 언어의 문장 구조 중 성격이 다른 하나는?**
1. while(expression) statement;
2. for(expression-1; expression-2; expression-3)statement;
3. if(expression) statement-1; else statement-2;
4. do {statement;} while(expression);

> **정답**: 3
> **해설**: 1번, 2번, 4번은 특정 조건에 따라 반복을 수행하는 루프 제어문(**반복문**)인 데 반해, 3번의 `if~else`는 프로그램 갈림길을 만드는 단순 **선택 제어문**입니다.

<br/>

**58. C 언어에서 생략되어서는 안되는 것은?**
1. 함수 main
2. 주석문
3. 출력문
4. 할당문

> **정답**: 1
> **해설**: C 언어는 프로그램의 구동 진입점이 되는 지점인 **`main` 함수** 정의가 필수적이어야만 돌아갑니다.

<br/>

**59. 페이지 교체 알고리즘 중 현 시점에서 가장 오랫동안 사용하지 않은 페이지를 교체하는 기법은?**
1. SCR
2. FIFO
3. LFU
4. LRU

> **정답**: 4
> **해설**: 참조된 시간을 기록해 두었다가 최근에 사용된 적이 가장 오래된 것('Least Recently Used')을 쫓아내는 전략이 **LRU**입니다. LFU는 '참조 횟수'가 가장 적은 것을 쫓아냅니다.

<br/>

**60. C 언어의 관계 연산자 중 ‘!=’의 의미는 무엇인가?**
1. 같다
2. 크다
3. 같지 않다
4. 작다

> **정답**: 3
> **해설**: 기호 `!`는 NOT(부정)의 의미를 가지므로, 등호와 합쳐진 `!=`는 양 변의 값이 서로 **다르다(같지 않다)**는 뜻이 됩니다. 같다는 `==`입니다.

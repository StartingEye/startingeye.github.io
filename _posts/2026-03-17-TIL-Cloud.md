## 📅 오늘 학습한 내용
AWS(Amazon Web Services)의 핵심인 컴퓨팅 인프라 구조를 이해하고, EC2 인스턴스를 직접 생성하여 리눅스 웹 서버를 구축하는 실습을 진행함.

---

## 1. AWS Core Concept
**리전(Region) & 가용 영역(AZ)**: 데이터 센터의 물리적 위치인 리전과 그 내부의 격리된 데이터 센터 집합인 AZ의 구조를 학습함.
**VPC(Virtual Private Cloud)**: 사용자만의 논리적으로 격리된 가상 네트워크 환경을 구성함.
**AMI(Amazon Machine Image)**: 인스턴스 실행에 필요한 OS와 소프트웨어 설정을 포함한 템플릿의 역할을 이해함.

---

## 2. EC2 실습 과정
```bash
# 1. AWS에서 지원하는 운영체제 접속
# 호스트: 퍼블릭 IP, 사용자: ec2-user, 인증: 개인키(.pem)

# 2. Apache 웹 서버 설치 및 실행 
sudo su                         # 루트 권한 획득
yum install httpd -y            # 아파치 설치
systemctl start httpd           # 서비스 시작

# 3. 웹 페이지 콘텐츠 작성 (index.html) 
cd /var/www/html
nano index.html
# "Welcome to MyWebserver(250506@ict.ac.kr)" 입력 후 저장
```
---

## 💡 느낀 점
* 보안 전문가를 목표로 공부하는 입장에서, 인스턴스의 송수신 트래픽을 제어하는 **보안 그룹(Security Group)** 설정이 서버 보안의 가장 기초적이면서도 강력한 방어선임을 깨달았습니다.
* 공개키는 서버에 저장하고 개인키는 로컬에 보관하는 **키 페어(Key Pair)** 방식의 자격 증명을 통해 클라우드 환경에서의 인증 보안 체계를 직접 경험해 볼 수 있었습니다.
* 실습 후 자원을 삭제하지 않으면 의도치 않은 비용이 발생할 수 있다는 점을 배워, 클라우드 자원 관리의 꼼꼼함이 필요함을 느꼈습니다.

## 💡 배운 내용
* **EC2 인스턴스 구조**: AMI와 인스턴스 유형이 결합되어 하나의 가상 서버가 생성되는 원리를 이해했습니다.
* **스토리지 차이**: 영구 저장소인 **EBS**와 임시 저장소인 **인스턴스 스토어**의 차이점 및 활용 방안을 배웠습니다.
* **서버 구축 프로세스**: Linux 환경에서 `yum` 패키지 매니저를 통해 Apache(`httpd`)를 설치하고 웹 서비스를 활성화하는 전체 과정을 습득했습니다.
* **모니터링**: CloudWatch를 통해 CPU 사용량 등 자원 상태를 감시하고 장애에 대비하는 방법을 파악했습니다.


## 앞으로 계획
* 이번에 배운 보안 그룹 설정을 응용하여, 특정 IP에서만 SSH 접속이 가능하도록 제한하는 등 더 정교한 **인바운드 규칙 설정**을 공부해 보겠습니다.
* Linux 명령어에 더 익숙해져서, 다음에는 사용자 데이터(User Data) 스크립트를 이용한 **서버 자동화 구축**도 시도해 볼 계획입니다.
* 정보보안학과 학생으로서 AWS의 다양한 보안 서비스(IAM, CloudTrail 등)를 추가로 학습하여 안전한 클라우드 아키텍처를 설계하는 역량을 키우겠습니다.
